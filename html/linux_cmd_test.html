基本用法

  1. 在左侧填写 SSH 主机 / IP、端口、用户名、密码
  2. 在「命令集合」中添加要远程执行的命令（支持预设模板、文件导入）
  3. 点击 ⚡ 测试连接 验证 SSH 连接是否正常
  4. 点击 ▶ 执行 后命令逐条流式执行；可用 暂停 / 恢复 控制是否继续接收结果（不中断远端已发起的命令）
  5. 右侧报告区展示每条命令的输出和退出码，可折叠查看
  6. 每条命令下方的备注在执行过程中即可点击输入；「编辑报告」用于修改命令输出正文
  7. 「↓ 下载」在浏览器本地下载报告；「💾 存档」打开确认框：默认文件名为 时间_客户端IP_SSH用户名.html（时间本地生成；客户端 IP 来自 /api/client-info；SSH 用户名为左侧表单用户名）；预览与存档成功后的访问链接为地址栏中的Web 服务器（如 服务器IP:8881/report/…）；与 SSH 目标主机无关；确认后写入 html/report/年月/
  8. 「📂 报告库」打开 reports.html，按月份浏览已存档的 HTML 报告

比对规则（= / ~ / !）

不在命令列表里设置。命令集合中只填写要执行的 shell 文本；=、~、! 仅用于与预期结果比对：

  • 在 对比视图 下，每条命令「实际输出」标题旁有三个小按钮：= 跟随全局「精确/模糊」、~ 强制模糊、! 忽略本条比对（显示「— 跳过」）
  • 点击按钮会写入内存中的预期数据；导出预期 JSON 时会把每条对应的 "marker" 字段一并保存（"" / "fuzzy" / "ignore"）
  • 也可直接编辑预期 JSON 中的 marker 后再 导入预期
版本号、时间戳等易变输出可在预期里设 "marker":"ignore"；含多余空格的输出可设 "fuzzy"。

预期结果比对

  1. 首次执行后点击 导出预期，生成含 cmd、output、可选 marker 的 JSON
  2. 再次执行后 导入预期，打开 对比视图 并排查看；用旁侧按钮调整 = / ~ / ! 后再次导出即可固化
  3. 每条命令显示「✓ 通过」「✗ 不符」或「— 跳过」
  4. 全局「模糊匹配 / 精确匹配」仅影响 marker 为空（=）的条目

命令文件格式（导入 / 导出 .txt）

每行一条原始命令，不含 ~ / ! 前缀（比对规则只在预期 JSON 里）：

# 注释行（以 # 开头）会被跳过
uname -a
hostname
df -h
^C              # 单独一行 ^C 表示插入「向远端发 SIGINT」的特殊行
  • 空行和 # 开头行跳过
  • 除 ^C 行外，整行原样作为命令文本

SSH 配置文件格式（导入 / 导出）

保存连接信息与纯命令列表。marker 仅用于 "ctrlc"（^C 行），不含 fuzzy/ignore：

{
  "version": 1,
  "host": "192.168.1.100",
  "port": 22,
  "user": "root",
  "pass": "password",
  "commands": [
    { "cmd": "uname -a" },
    { "cmd": "kill -INT $$", "marker": "ctrlc" }
  ]
}
  • 导出配置：连接 + 命令（^C 行带 marker:"ctrlc"）
  • 导入配置：填充表单与命令列表

注意事项

  • 服务器需已安装并运行 openssh-server，且允许密码认证
  • 每条命令独立建立 SSH 连接，超时时间为 15 秒
  • 命令执行顺序与列表顺序一致，逐条串行执行
  • 退出码 0 表示成功，非零表示失败或异常
  • 输出内容上限为 512 KB / 条，超出部分截断

确认存档报告

将保存到服务器目录 html/report/—/（月文件夹以服务器本地时间为准；下方链接中的年月按浏览器本机日历预览，跨天/时区时可能与实际目录差一月）。

默认文件名：时间_客户端 IP_SSH 用户。IP 来自 —（GET /api/client-info，TCP 对端）；SSH 用户来自左侧表单「用户名」当前值（默认 root）。

预览与存档成功后的访问链接使用 Web 服务器（与地址栏一致）：

示例：http://10.150.72.102:8881/report/202603/文件名.html — 主机与端口随当前页 URL。
