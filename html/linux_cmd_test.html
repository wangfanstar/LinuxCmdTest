基本用法

  1. 在左侧填写 SSH 主机 / IP、端口、用户名、密码
  2. 在「命令集合」中添加要远程执行的命令（支持预设模板、文件导入）
  3. 点击 ⚡ 测试连接 验证 SSH 连接是否正常
  4. 点击 ▶ 执行 后命令逐条流式执行；可用 暂停 / 恢复 控制是否继续接收结果（不中断远端已发起的命令）
  5. 右侧报告区展示每条命令的输出和退出码，可折叠查看
  6. 执行报告终端窗口内底部的报告备注用于整份说明/结论；每条命令下方的备注在执行过程中即可点击输入；「编辑报告」用于修改命令输出正文
  7. 「↓ 下载」在浏览器本地下载报告；「💾 存档」打开确认框：默认文件名为 时间_客户端IP_SSH用户名.html；确认后写入服务器 html/report/<SSH用户名目录>/<服务器年月>/（用户名由左侧「用户名」净化；年月为服务器本地 YYYYMM）。报告工具栏「导入预期」旁：清空配置、读档配置、存档配置（与「💾 存档」报告类似，弹窗可改文件名后写入同目录规则下的 .json；读档从该用户目录列出并加载）。客户端 IP 来自 /api/client-info；访问链接使用地址栏 Web 服务器（与 SSH 目标无关）
  8. 「📂 报告库」打开 reports.html，按月份浏览已存档的 HTML 报告

比对规则（≈ / ! / #）

不在命令列表里设置。命令集合中只填写要执行的 shell 文本；对比视图下的 ≈、!、# 仅用于与预期结果比对：

  • 在 对比视图 下，每条命令「实际输出」标题旁：≈ 为「忽略空格」开关（开：按空白折叠后比对，等同 JSON marker:"fuzzy"；关：跟随全局「精确/模糊」）；! 忽略本条比对（显示「— 跳过」）；# 切换是否忽略以 # 开头的行
  • 点击按钮会写入内存中的预期数据；导出预期 JSON 时会把每条对应的 "marker" 字段一并保存（"" / "fuzzy" / "ignore"）
  • 也可直接编辑预期 JSON 中的 marker 后再 导入预期
版本号、时间戳等易变输出可在预期里设 "marker":"ignore"；需忽略空格差异时打开 ≈ 或设 "marker":"fuzzy"。

预期结果比对

  1. 首次执行后点击 导出预期，生成含 cmd、output、可选 marker 的 JSON
  2. 再次执行后 导入预期，打开 对比视图 并排查看；用旁侧按钮调整 ≈ / ! / # 后再次导出即可固化
  3. 工具栏「显示预期」在已加载预期时启用：若曾点到「单结果视图」，可用它一键恢复并排比对；若尚无报告区内容，需先执行命令生成报告
  4. 每条命令显示「✓ 通过」「✗ 不符」或「— 跳过」
  5. 全局「模糊匹配 / 精确匹配」仅影响 ≈ 关闭且 marker 非 fuzzy、非 ignore 的条目

命令文件格式（导入 / 导出 .txt）

每行一条原始命令，不含比对前缀（比对规则只在预期 JSON / 对比视图按钮里）：

# 注释行（以 # 开头）会被跳过
uname -a
hostname
df -h
^C              # 单独一行 ^C 表示插入「向远端发 SIGINT」的特殊行
  • 空行和 # 开头行跳过
  • 除 ^C 行外，整行原样作为命令文本

SSH 配置文件格式（导入 / 导出）

保存连接信息与纯命令列表。marker 仅用于 "ctrlc"（^C 行）；skipSeq 表示该行在侧栏不参与显示序号（可选）：

{
  "version": 1,
  "host": "192.168.1.100",
  "port": 22,
  "user": "root",
  "pass": "password",
  "commands": [
    { "cmd": "uname -a" },
    { "cmd": "注释说明用", "skipSeq": true },
    { "cmd": "kill -INT $$", "marker": "ctrlc" }
  ]
}
  • 导出配置：连接 + 命令（^C 行带 marker:"ctrlc"；不计序号行带 skipSeq:true）
  • 导入配置：填充表单与命令列表；未包含 pass / password 时保留当前密码框内容。"pass":null 或空字符串表示清空（需重填）。亦支持 password 字段名（与 pass 二选一）
  • 侧栏每条命令前的序号可点击切换为「–」不计序号，仅影响展示，执行顺序不变

注意事项

  • 服务器需已安装并运行 openssh-server，且允许密码认证
  • 每条命令独立建立 SSH 连接，超时时间为 15 秒
  • 命令执行顺序与列表顺序一致，逐条串行执行
  • 退出码 0 表示成功，非零表示失败或异常
  • 输出内容上限为 512 KB / 条，超出部分截断
