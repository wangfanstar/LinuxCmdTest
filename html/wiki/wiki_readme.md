# Wiki 使用说明

本文档说明 `notewiki` 的日常使用方法，包括登录权限、文章编辑、附件上传、导出功能和用户管理。

## 1. 入口页面

- Wiki 编辑页：`/wiki/notewiki.html`
- Wiki 管理页（管理员）：`/wiki/wiki-auth-admin.html`
- 文章索引页：`/wiki/notewikiindex.html`

## 2. 权限模型

系统有 3 类权限：

- `admin`：所有权限（含用户管理）
- `author`：可进行文章编辑、上传、发布、删除等 Wiki 操作（不含用户管理）
- `guest`：只读浏览权限

未登录用户默认按游客处理，只能浏览。

## 3. 登录与状态

- 在 `notewiki.html` 右上角点击“登录”进入登录弹窗。
- 登录成功后，右上角会显示当前账号与角色。
- 登录状态通过会话保存，刷新页面后仍生效（会话有效期内）。

## 4. 文章编辑流程

1. 在左侧点击 `✚ 新建` 创建文章，填写标题、分类（可选）。
2. 进入编辑区后输入 Markdown 内容。
3. 点击“保存”或使用 `Ctrl+S` 保存。
4. 保存后会自动生成对应 HTML 页面。

说明：

- 编辑时会做权限校验，`guest` 无法执行写操作。
- 每次保存会记录修改历史到 SQLite（用于后续查询）。

## 5. 代码块与语法高亮

使用 Markdown 代码块语法：

```markdown
```c
int main() { return 0; }
```
```

- 预览区会显示代码高亮。
- 发布后的 HTML 页面也会显示代码高亮。

## 6. 图片与附件上传

- 上传图片：工具栏图片按钮
- 上传附件：工具栏附件按钮

快捷键：

- `Ctrl + Alt + I`：插入图片
- `Ctrl + Alt + A`：插入附件

上传后会自动插入 Markdown 引用链接。

## 7. 导出功能

- 导出 MD ZIP：导出当前文章 Markdown 及关联资源
- 导出 PDF：支持目录书签与页码（依赖服务端 PDF 工具链）

## 8. 用户管理（管理员）

进入 `wiki-auth-admin.html` 可进行：

- 新增用户
- 修改用户（用户名/密码/权限/分组）
- 删除用户
- 查看操作日志（默认前 10 条）
- 查看 MD 修改历史（默认前 10 条）
- 查看用户文章数量排名

说明：

- 管理页不显示 Admin 账号。
- Admin 账号信息来自 `sqlite_db/db.config` 配置文件。

## 9. Admin 配置文件

路径：`/wiki/sqlite_db/db.config`

示例：

```ini
admin_username=Admin
admin_password=123456
```

服务启动时会读取该文件并应用管理员账号配置，Admin 默认始终拥有管理员权限。

## 10. 常见问题

- 登录提示无权限：检查账号角色是否为 `author` 或 `admin`。
- 无法编辑：当前可能是 `guest` 或未登录。
- 管理页打不开：仅 `admin` 可访问。
- 修改 Admin 配置未生效：重启服务后再试。
