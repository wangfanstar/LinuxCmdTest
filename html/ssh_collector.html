使用帮助 · Linux 命令行测试工具

基本用法

  1. 在左侧填写 SSH 主机 / IP、端口、用户名、密码
  2. 在「命令集合」中添加要远程执行的命令（支持预设模板）
  3. 点击 ▶ 执行 按钮，等待命令逐一执行
  4. 右侧报告区展示每条命令的输出和退出码，可折叠查看
  5. 点击「编辑报告」可修改输出内容或添加备注
  6. 点击「↓ 下载」将报告保存为独立 HTML 文件

命令文件格式（导入文件）

支持上传 .txt / .sh / .conf 纯文本文件，每行一条命令：

# 这是注释，会被跳过
uname -a
hostname
uptime

# 磁盘与内存
df -h
free -h

解析规则：

  • 空行自动跳过
  • 以 # 开头的行视为注释，自动跳过
  • 导入的命令追加到当前列表末尾，不会覆盖已有命令

SSH 配置文件格式（导入 / 导出）

配置文件为 .json 格式，保存连接信息和命令列表。密码不会写入文件。

{
  "version": 1,
  "host": "192.168.1.100",
  "port": 22,
  "user": "root",
  "commands": [
    "uname -a",
    "df -h",
    "free -h"
  ]
}
  • 导出配置：将当前表单中的主机、端口、用户名和命令列表保存为 JSON 文件下载
  • 导入配置：选择 JSON 文件后自动填入连接字段并替换命令列表
导入配置后密码字段不会自动填写，需手动输入密码后再执行。

注意事项

  • 服务器需已安装并运行 openssh-server，且允许密码认证
  • 每条命令独立建立 SSH 连接，超时时间为 60 秒
  • 命令执行顺序与列表顺序一致，逐条串行执行
  • 退出码 0 表示成功，非零表示失败或异常
  • 输出内容上限为 512 KB / 条，超出部分截断
