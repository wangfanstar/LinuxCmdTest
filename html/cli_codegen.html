③ arg_id 编号规则

回调函数中 cli_cmd_get_arg(ctx, arg_id, …) 的 arg_id 按如下规则分配：

示例模板（CMD_REG 展开后的实际形态）：
[ chip INTEGER ] 框架自动跳过 ▶ llr enable block INTEGER<0-15>
arg_id=0 port-mask HEX<0-FF>
arg_id=1

规则 1：关键字不占 arg_id

只有参数类型 token（INTEGER/HEX/STRING 等）和选择块才占 arg_id，纯关键字不占。

规则 2：chip 前缀不计入

CMD_REG 追加的 [ chip N ] 框架内部处理，业务代码 arg_id 从 0 开始取业务参数，不需要跳过 chip。

规则 3：选择块算 1 个 arg_id

整个 {a|b|c} 块共享 1 个 arg_id，取到的值是所选项的索引（0/1/2）。

规则 4：可选块判断返回值

可选块内的参数有 arg_id，但若用户未提供，cli_cmd_get_arg 返回非 GT_SUCCESS，用返回值判断有无，不要强制校验。

④ cmd_descs 对齐规则

cmd_descs 中每一行（以 \n 结尾）对应 cmd_str 中的一个有意义的 token，顺序必须完全一致。

不计入行数的 token：[ ] { } |（结构符号不需要说明行）。

cmd_str vs cmd_descs 对应关系：
show→ CMD_HELP_SHOW_STR（特殊宏，含换行）
llr→ "Llr keyword\n"
enable→ "Enable keyword\n"
[→ 不需要说明行
verbose→ "Enable verbose output\n"
]→ 不需要说明行
block→ "Block keyword\n"
INTEGER<0-15>→ "MAC block index (0-15)\n"

⑤ View（视图）说明

View 是命令的作用域，用户进入某个 View 后只能执行该 View 内注册的命令（加上 PUBLIC 全局命令）。
注册时通过 CMD_REG(CMD_VIEW_XXX, &cmd) 指定目标 View。

View 常量ID用途典型命令
CMD_VIEW_PUBLIC0始终可见，所有用户均可访问show version / help / quit
CMD_VIEW_SYSTEM1系统级调试，需要高权限进入系统参数配置、内存转储
CMD_VIEW_DEBUG2寄存器/状态机调试（最常用）llr enable / show llr status
CMD_VIEW_SHELL3Shell 风格操作命令文件操作、脚本执行
CMD_VIEW_TEST4自动化测试套件run test / report result
CMD_VIEW_IF5接口配置（Interface 模式）port config speed / vlan add
CMD_VIEW_SLT6Silicon Life Test（量产测试）slt run / slt result
CMD_VIEW_UT7单元测试（Unit Test）ut run module / ut report
CMD_VIEW_HGSI8高速 GPIO 接口调试gpio set / gpio read

⑥ 完整写法示例对照

以下展示 4 种典型命令模式及对应 C 代码结构：

▌ 模式 A：纯关键字命令
show version
→ 无参数，chip 由 cli_cmd_get_chip(ctx) 获取，直接实现逻辑
▌ 模式 B：带整数 + 十六进制参数
llr enable block INTEGER<0-15> port-mask HEX<0-FF>
arg_id=0 → block (uint8，max=15≤255自动推断)
arg_id=1 → port_mask (uint32，HEX固定uint32)
▌ 模式 C：含可选关键字
show counter block INTEGER<0-15> [ verbose ]
arg_id=0 → block
verbose 是可选关键字：bool has_verbose = (cli_cmd_get_arg(ctx, 1, &tmp, sizeof(tmp)) == GT_SUCCESS);
▌ 模式 D：含选择块
port speed { 10G | 25G | 100G | 400G }
arg_id=0 → speed 索引（10G=0, 25G=1, 100G=2, 400G=3）
回调里：const char *names[] = {"10G","25G","100G","400G"}; names[speed]

⑦ 常见错误速查

❌ cmd_descs 行数不对

每个有意义 token（不含 [ ] { } |）必须有对应的一行描述，多一行少一行都会导致 help 显示错乱。

❌ 忘记 GT_STRUCT_ZERO

注册多条命令时，每条命令前必须调用 GT_STRUCT_ZERO(cmd)，否则上条命令的 cmd_title/api 字段会残留。

❌ arg_id 从 1 开始

CMD_REG 的 chip 前缀不占 arg_id，业务参数从 arg_id=0 开始取，不需要跳过任何东西。

❌ 用 printf 而非 cli_printf

所有输出必须用 cli_printf(ctx, …)，直接 printf 的输出只到服务进程 stdout，Socket 客户端看不到。

❌ BYTESTR 奇数长度

BYTESTR 严格要求偶数位十六进制（每2字符=1字节），输入奇数位会报 CMD_PARSE_LEN_NOT_EVEN 错误。

❌ HEX 范围用十进制写

HEX<0-255> 是错误写法，范围值应用十六进制：HEX<0-FF>。框架按十六进制解析范围值。
