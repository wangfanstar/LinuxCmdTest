C Header → YAML 转换工具 · 使用指南

快速上手

  • 将 .h 文件拖到左侧 "上传区"，或粘贴 C 代码，点击 ▶ 转换
  • 也可直接在页面任意空白处 Ctrl+V 粘贴并自动转换
  • 右侧 YAML 支持 实时编辑，可与自动生成内容混合修改
  • "手动" 标签页可在不写代码的情况下追加 Enum / Object / Attribute

自动提取的 C 元素

C 语法→ YAML 节说明
#define NAME VALUEmacros → fields相邻宏合并为一组；跳过保护宏与函数宏
typedef enum { ... } name_t;enums提取枚举值及行尾 ///< 注释
typedef struct { ... } name_t;structs提取字段名、类型、注释；数组字段 uint8 x[8] 自动拆分为 type: uint8 + array_size: 8（模板必需格式）；自动生成 c_type: gt_{name}_t 和 m_type: {name}
typedef union { ... } name_t;unions同上，同样自动补全 c_type / m_type，数组字段同样拆分
typedef BASE new_t;defs类型别名
gt_status_t func(...);注释行勾选"函数声明提示"后以注释形式列出

struct / union 必需字段

production autogen.py 在 collect_global_data 中直接访问 s['c_type']，没有默认值回退，缺失时会抛出 KeyError: 'c_type'。

字段含义示例
c_type生成的 C 类型名（加入 all_struct_names 集合）gt_rlm_fsm_t
m_typeunion member 访问名（type_map 使用）rlm_fsm

本工具自动生成规则：c_type = {前缀}{name}_t，前缀默认 gt_，可在"选项"面板修改。

attribute flags 参考

Flag含义
READ_ONLY只读，只能 get
CREATE_AND_SET创建时可指定，创建后可修改
CREATE_ONLY仅创建时指定，之后只读
MANDATORY_ON_CREATE创建时必须提供

命名规则

  • 枚举/结构名自动去除前缀（默认：ql_npu_ ql_ gt_）
  • 去除类型后缀 _t / _e / _s
  • 可在"选项"面板自定义前缀列表

基准 YAML — 多文件合并去重

点击顶栏 ⚓ 基准 YAML 加载一个已有的 .yaml 文件作为基准。之后每次转换时：

  • 新文件中有、基准中没有的定义 → 直接追加
  • 两者均有且内容相同 → 保留基准版本（保留手动编辑的内容）
  • 两者均有但内容不同（冲突） → 弹出对比框，逐一选择：
    • 保留基准 — 丢弃新版，保留原有
    • 使用新版 — 以新解析结果覆盖
    • 手动编辑 — 展开编辑框自由修改后确认
    • 全部保留基准 — 一键跳过所有冲突，全保留基准
  • 基准中有、新文件中没有的定义 → 保留（不会丢失已有内容）
  • 基准的 objects: 节 → 始终完整保留，不参与冲突检测

点击 ✕ 基准 可清除基准，恢复普通转换模式。

显示 / 隐藏默认值字段

工具默认对每个条目生成 所有可选字段，包括 brief: ""、dep_level: 0、array_size: ~ 等，方便直接作为模板填写。

点击右上角 隐藏默认值 按钮可切换为 精简模式：凡值等于默认值的可选字段均不输出，YAML 更简洁。

字段默认值隐藏条件
dep_level0值为 0 时隐藏
brief""无注释时隐藏
value (enum field)~C 头文件中未显式赋值时隐藏
array_size~非数组字段时隐藏

切换模式后自动重新转换，不影响已手动编辑的 YAML 内容（因为会重新生成）。

快捷键

  • Ctrl+V（空白处）— 粘贴并立即转换
  • Ctrl+Enter（在输入框内）— 触发转换
  • Ctrl+Z（在 YAML 编辑器内）— 撤销
  • Ctrl+Y / Ctrl+Shift+Z（在 YAML 编辑器内）— 重做
  • Escape — 关闭帮助
